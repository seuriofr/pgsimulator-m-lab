PG模拟器 · 功能与场景

面向开发与测试场景的轻量级模拟环境，降低依赖成本，提升迭代效率。

站点背景 v1.2

PG模拟器 最初为个人项目孵化，后逐步整理为公开工具站点。它并非完整的数据库发行版，而是提供一组贴合 PostgreSQL 行为模式的模拟接口与数据构造能力。

团队在早期集成测试中频繁遇到临时数据库搭建慢、环境不一致的问题，因此采用行为模拟的思路来缩短准备时间。站点内的核心模块覆盖了常见查询模式与返回结构，并持续根据社区反馈校准语义。

基于 PG 14 / 15 行为参考

可信说明 透明度

站内所有模拟逻辑均以公开的 PostgreSQL 文档与标准 SQL 行为为基准构建，不包含闭源黑盒组件。每个接口的返回结构、数据类型映射和错误提示都附有简短参考说明。

模拟器不会持久化用户传入的查询或数据，所有交互仅在当前会话上下文中计算。代码仓库的部分模块以开源形式发布，便于同行审查与改进。

  • 类型映射参考自 PG 系统表
  • 事务隔离等级模拟为 Read Committed
  • 支持常见 JSON / 数组 / 范围类型

适用场景 轻量·快速

适用于前端原型联调、单元测试的数据层 Mock、CI 流水线中无法快速拉起真实 PG 实例的环境。也可作为学习基础 SQL 行为时的一种轻交互辅助。

对于生产环境或需要严格 ACID 保障的用例，仍建议使用官方 PostgreSQL 实例。模拟器在语法覆盖上聚焦于 DML 和常见查询模式，不涉及物理复制、WAL 或自定义函数扩展。

若需体验完整功能，可直接试用站点中的在线沙盒。

想要进一步了解模拟器的设计思路与操作方式？可以前往 PG模拟器 查阅交互说明与接口文档。

专题页仅作概要介绍，详细用例与版本日志均在主站持续更新。
